PROJECT HISTORY - CHECKPA19

Purpose
- Native Android checklist app for cockpit-style use.
- Kotlin + Jetpack Compose.
- Single-screen checklist flow, offline only.
- Static JSON checklist in assets.
- SharedPreferences used for session restore.

Current Project Location
- /home/olivier/checkpa19

Repository Status
- Git repository initialized locally.
- Current main branch: main

Functional Scope
- Aircraft: PA19
- Checklist phases: 1 to 10
- Ground use only
- Offline only

Important Product Decisions
- No over-engineering.
- No network.
- No database.
- No multi-screen navigation for the checklist itself.
- Priority is instant understanding of the current checklist position.
- One-hand usage and visual clarity are more important than visual richness.

Checklist Behavior Implemented
- Linear progression only.
- 1 tap = 1 validation.
- Automatic move to next item.
- Automatic move to next phase.
- Automatic scroll to keep current item visible.
- Exact restore after app close using SharedPreferences.
- Keep screen on during checklist.
- Pause button disables keep-screen-on.

Current UX Flow
1. App start at beginning of a new session:
   - Full PA19 memo screen is displayed first.
   - Button: COMMENCER
2. Checklist flow:
   - Current phase shown at top.
   - Current item visually dominant.
   - Done items greyed.
   - Next items neutral.
3. End of checklist:
   - Only the speed/reference memo is shown.
   - Button: TERMINER
4. TERMINER resets session and returns to the start memo for the next run.

Memo Screen Decisions

- End-of-checklist screen displays only speed/references, not the full memo.

Visual Decisions
- Keep the UI visually restrained.
- Avoid adding too many colored backgrounds in checklist rows.
